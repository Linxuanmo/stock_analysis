# 下一季度需求：参数训练与管理系统

## 1. 需求背景
当前系统已具备数据获取、特征工程（高峰向量切割、整体时序分析）和策略生成基础框架，但所有分析参数（如高峰显著度、片段长度、波段阈值、特征权重等）均以常量形式硬编码在代码中，调整效率低，且无法利用历史数据自动寻找更优参数组合。同时，系统缺乏对训练好的参数或模型进行版本管理、复用和切换的能力。因此，下一季度需要建设一套**可训练、可管理、可选择的参数学习和复用体系**，实现：
- 通过深度学习或优化算法自动学习最优的切割参数和特征权重；
- 提供手动配置和自动加载训练结果两种参数使用模式；
- 所有训练结果均带时间戳存储，支持历史版本回溯与对比。

## 2. 功能需求

### 2.1 训练模式
1. **两类训练目标**：
   - **基本向量切割参数的默认值**：训练模型学习如何从原始价格序列中输出最优的 `PROMINENCE`、`MAX_SEGMENT_LEN` 等参数。
   - **特征向量的权重**：针对已固定的切割参数，训练模型输出多个特征（长度、波动率、斜率等）在策略评分中的加权系数。
2. **模型选择**：允许从已注册的模型库（TensorFlow/PyTorch/Jittor/手写模型）中自由选择训练器。
3. **数据范围自由选择**：用户可以指定使用哪部分股票、哪个时间段的数据进行训练，另一部分数据可选择沿用某次历史训练的默认结果（增量训练或迁移学习）。
4. **训练结果保存**：每次训练完成后，自动记录时间戳，将模型文件和参数结果存入对应模块的模型文件夹中（例如 `trained_models/cutting_params/20260604_120000/`）。

### 2.2 使用模式
1. **参数管理**：为“切割参数默认值”和“特征权重”分别建立管理文件，允许：
   - **手动模式**：直接编辑管理文件中的数值，所有分析模块自动读取最新配置。
   - **自动模式**：从指定时间戳的训练结果中自动加载参数，实现一键切换。
2. **版本记录**：每次训练或手动修改后，均生成带时间戳的版本快照，便于对比和回溯。
3. **统一接口**：特征分析器（`image_peak_analyzer.py`、`sequential_analyzer.py`）初始化时通过统一的参数管理器获取当前生效的参数，无需关心来源。

## 3. 技术方案

### 3.2 核心模块说明
- **`param_trainer.py`**：以原始价格序列或特征表为输入，使用深度学习模型（LSTM/CNN/Transformer）学习最优切割参数，或采用贝叶斯优化直接搜索参数空间。
- **`weight_trainer.py`**：接收特征向量表与回测收益标签，训练线性模型、树模型或小型神经网络，输出特征权重系数。
- **`param_optimizer.py`**：将参数组合送入回测引擎，返回目标函数值（夏普比率、最大回撤等），供训练器调用。
- **`param_manager.py`**：核心管理类，负责：
  - 从手动预设文件或自动训练结果中读取参数
  - 向分析器提供当前生效的参数
  - 保存新训练结果（自动创建时间戳目录）
  - 维护参数版本历史
- **`preset_cutting_params.py` / `preset_feature_weights.py`**：以“配置区+注释”风格提供手动编辑的默认值，可被自动模式覆盖。
- **`training_service.py`**：封装完整训练流程，用户通过界面或命令行选择模型、数据范围和历史基准，调用训练器并触发保存。

### 3.3 数据流
**训练模式**：  
用户选择数据范围与模型 → `training_service` 编排流程 → `param_trainer`/`weight_trainer` 训练 → `param_optimizer` 调用 `backtest.py` 评估 → 结果存入 `trained_models/` 带时间戳的子目录。

**使用模式**：  
分析器初始化时 → 从 `param_manager` 获取当前模式（手动/自动）及参数值 → 若为手动，读取 `preset_*.py`；若为自动，从指定时间戳的 `trained_models/` 目录加载 → 分析器使用参数执行特征工程。

## 4. 实施计划
1. **基础建设**（优先级最高）  
   实现 `param_manager.py`、`preset_cutting_params.py`、`preset_feature_weights.py`，并修改 `image_peak_analyzer.py` 和 `sequential_analyzer.py` 使其通过管理器获取参数，同时支持手动编辑和从文件加载。
2. **训练能力**  
   开发 `param_optimizer.py`（连接回测引擎），再开发 `param_trainer.py` 和 `weight_trainer.py`，至少实现一种优化算法（建议先做贝叶斯优化以降低调试复杂度）。
3. **流程与存储**  
   实现 `training_service.py`，定义 `trained_models/` 的存储规范，实现自动保存和时间戳命名。
4. **配置与注册**  
   扩展 `config/default.yaml`、`optimization.yaml` 和 `model_registry.yaml`，完成模型注册与训练参数管理。
5. **测试与文档**  
   编写测试脚本，验证手动/自动切换、训练保存、分析器正确读取参数等流程。

通过以上建设，系统将具备数据驱动的自适应能力，同时保留灵活的人工干预接口，为后续的策略优化和AI辅助决策打下坚实基础。
