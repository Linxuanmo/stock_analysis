# 数据读取标准方法说明

> 适用版本：v1.0  
> 最后更新：2026-06-07  
> 维护者：项目开发团队

## 1. 数据来源与格式要求

### 1.1 来源
- **东方财富/通达信金融终端**：手动导出历史K线数据。
- **AKShare**：在线自动下载（可选）。

### 1.2 导出设置
| 设置项       | 推荐值                         | 说明                             |
|--------------|--------------------------------|----------------------------------|
| 数据粒度     | 日线                           | 也可导出周线、月线，但文件名需区分 |
| 复权方式     | 前复权                         | 保持价格连续性                   |
| 导出格式     | 文本文件（制表符分隔）         | 导出后文件后缀通常为 `.xls`，实为文本 |
| 导出内容     | OHLCV + 常用指标（MA, MACD等） | 只需保留价格与成交量，其余列会被忽略 |
| 文件命名     | `序号_股票代码.xls`            | 例如 `1_688469.xls`，序号可随意  |
| 存放位置     | `data/raw/` 下的任意子文件夹   | 支持多级子目录，所有 `.xls` 将被自动扫描 |

### 1.3 对文件内部格式的假设
- 编码：**GBK**（通达信中文环境默认）
- 分隔符：**制表符 `\t`**
- 表头行：**第2行**（第1行为股票名称或空行，第2行为列名 `时间 开盘 最高 最低 收盘 成交量 ...`）
- 注释行：以 `#` 开头（如文件末尾的 `#数据来源:通达信`），会被自动忽略。

## 2. 数据读取标准流程

### 2.1 使用项目内置读取器 `OrientalFetcher`
所有数据读取均通过 `core/data/oriental_fetcher.py` 完成，无需手动处理格式差异。

### 2.2 基本用法
```python
from core.data.oriental_fetcher import OrientalFetcher

# 创建读取器（默认读取 data/raw/）
fetcher = OrientalFetcher()

# 加载单只股票
df = fetcher.load_stock('688469')   # 返回清洗后的DataFrame

# 加载所有股票（字典）
all_stocks = fetcher.load_all_stocks()  # {代码: DataFrame}

# 合并为一张大表
big_df = fetcher.load_all_stocks_as_dataframe()
