# 特征向量说明文档

> 适用项目：科创板股票分析系统  
> 对应数据文件：`data/processed/peak_features.csv`、`data/processed/sequential_features.csv`  
> 最后更新：2026-06-07

## 1. 视角A – 高峰片段特征 (`peak_features.csv`)

每一行代表**一个高峰切割片段**（由视角A分析器产生）。

| 列名 | 数据类型 | 含义与计算方法 | 用途举例 |
|------|----------|----------------|----------|
| `code` | 字符串 | 股票代码（如 688469） | 区分不同股票 |
| `segment_id` | 整数 | 该股票内片段的编号（从 0 开始） | 按时间排序的片段顺序 |
| `start_date` / `end_date` | 日期 | 片段的起始和结束日期 | 对照K线图定位该片段 |
| `length` | 整数 | 片段包含的交易日数 | 判断波动周期的长短 |
| `price_change` | 浮点数 | 片段首尾价格变化率：`(末尾价 - 起始价) / 起始价` | 正值为上涨片段，负值为下跌片段 |
| `max_price` / `min_price` | 浮点数 | 片段内的最高价和最低价 | 衡量波动范围 |
| `volatility` | 浮点数 | 片段内日收益率的标准差 | 波动越大，风险或活跃度越高 |
| `avg_volume` | 浮点数 | 片段内日均成交量 | 观察放量/缩量情况 |
| `trend_slope` | 浮点数 | 线性趋势斜率：`(末价 - 首价) / length` | 斜率越大，趋势越急 |
| `normalized_series` | 文本（数组） | 片段收盘价序列归一化到 [0,1] 的结果，存储为字符串（如 `[0.12 0.25 ... 0.89]`） | 用于比较不同片段的形态相似度（可直接转回 numpy 数组） |

**使用建议**：
- 先用 `price_change`、`length`、`volatility` 等统计特征做初步筛选（如只保留上涨片段、波动适中的片段）。
- 再用 `normalized_series` 计算片段之间的欧氏距离或 DTW 距离，找出历史上与当前片段最相似的形态。

## 2. 视角B – 整体时序特征 (`sequential_features.csv`)

每一行代表**一只股票的整体统计特征**。

| 列名 | 数据类型 | 含义与计算方法 | 用途举例 |
|------|----------|----------------|----------|
| `code` | 字符串 | 股票代码 | 区分不同股票 |
| `total_days` | 整数 | 该股票数据的总交易日数 | 判断数据量是否足够 |
| `start_date` / `end_date` | 日期 | 数据覆盖的起止日期 | 了解时间跨度 |
| `overall_return` | 浮点数 | 整体收益率：`(最新价 - 首日价) / 首日价` | 判断总体涨跌 |
| `volatility` | 浮点数 | 整个序列的日收益率标准差 | 整体波动水平 |
| `up_wave_count` / `down_wave_count` | 整数 | 上升波段和下降波段的数量 | 上升波段多说明多头更主动 |
| `up_wave_avg_days` / `down_wave_avg_days` | 浮点数 | 上升/下降波段的平均持续天数 | 判断趋势的持续性 |
| `up_wave_avg_pct` / `down_wave_avg_pct` | 浮点数 | 上升/下降波段的平均涨跌幅 | 平均每波力度 |
| `momentum` | 浮点数 | 最新收盘价相对于 `MOMENTUM_MA_PERIOD` 日均线的偏离百分比 | 正值→短期偏强，负值→偏弱 |
| `trend_state` | 字符串 | 当前趋势状态：`above`（价格持续在均线上方）、`below`（持续在下方）、`mixed`（无明显倾向） | 判断当前属于买方主场还是卖方主场 |
| `consecutive_days_vs_ma` | 整数 | 最新价格持续在均线上方或下方的天数 | 趋势的坚定程度 |
| `avg_volume` | 浮点数 | 日均成交量 | 活跃度参考 |
| `volume_trend` | 浮点数 | 成交量变化率：`(最新成交量 - 最旧成交量) / 最旧成交量` | 放量或缩量趋势 |

**使用建议**：
- 可根据 `trend_state` 和 `momentum` 快速判断股票当前是多头主导还是空头主导。
- 结合 `up_wave_count` 和 `down_wave_count` 的比值，评估股票的历史趋势偏好。
- 用于板块内部横向对比，寻找趋势最健康或反转概率最高的个股。
